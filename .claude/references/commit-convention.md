# 커밋 메시지 규칙

## 형식

```
<type>: <한글 제목>

- 변경 내용 1
- 변경 내용 2
```

## 규칙

1. **type**: 영문 소문자, 아래 목록 중 하나
2. **제목**: 한글로 작업 내용 한 줄 요약 (영문 금지)
3. **빈 줄**: 제목과 본문 사이 반드시 빈 줄
4. **본문**: `- ` 로 시작하는 항목 나열 (선택, 있을 경우 형식 준수)

## 허용 type

| type | 용도 |
|------|------|
| `feat` | 새 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 변경 |
| `style` | 포매팅, 세미콜론 등 (코드 변경 없음) |
| `refactor` | 리팩토링 |
| `test` | 테스트 추가/수정 |
| `chore` | 빌드, 설정, 패키지 등 |
| `perf` | 성능 개선 |
| `ci` | CI/CD 설정 |
| `build` | 빌드 시스템 변경 |
| `revert` | 이전 커밋 되돌리기 |

## 예시

```
feat: 세션 분석 API 엔드포인트 추가

- GET /sessions/:id 구현
- 비효율 패턴 검출 로직 연동
- Zod 스키마 검증 추가
```

```
fix: OTLP 수신 포트 설정 오류 수정

- 환경변수 OTLP_HTTP_PORT 누락 처리 추가
```

```
chore: Turborepo 설정 및 의존성 초기화
```

## 검증

커밋 시 `.claude/scripts/validate-commit.js` hook이 자동으로 형식을 검증한다.
형식이 맞지 않으면 커밋이 차단된다.
